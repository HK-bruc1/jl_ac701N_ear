## 1. 产品、性能和应用简介

### 1.1 产品说明

芯存SD NAND是一种嵌入式存储芯片，采用SD接口技术，应用与SD卡/TF卡/Micro SD卡类似。SD NAND的优势包括体积小、高品质、低功耗以及高性价比。

### 1.2 性能指标

- 支持SD系统规范: SD2.0/SD1.1/SD1.0
- 支持容量: 1GBit/2GBit/4GBit
- 读写速度: 最高支持到C6读写速度
- 外观尺寸: 1GBit/2GBit/4GBit: 8x6mm
- 内置8bit/512Bytes ECC校验和纠错
- 供电电压: 3.3V
- 支持最高工作频率: 50MHz
- 总线位宽: 8位
- 集成片上POR(上电复位)和BOR(欠压复位)
- 集成片上稳压器
- 支持空闲(IDLE)和睡眠(SLEEP)两种省电模式
- 支持提高数据存储和处理的效率的功能: 缓存、命令队列、坏块管理、智能垃圾回收、支持交错、缓存、多平面编程
- 工作温度: -25°C至85°C

## 2. 参考设计

### 电路设计注意事项

1. **RDAT** 和 **RCMD** (10K~100kΩ)是上拉电阻，用于在SD NAND处于高阻态时防止CMD和DAT线路总线悬浮。
2. 主机应通过RDAT将所有DAT0-3线路进行上拉，即使主机在SD模式下仅将SD NAND用作1位模式也是如此。
3. 建议在VCC上连接2.2uF的电容。
4. **RCLK**的参考阻值为0~120Ω。

### 引脚连接

- **DAT2**: 数据位2，通过10KΩ上拉电阻连接3.3V
- **DAT3**: 数据位3，通过10KΩ上拉电阻连接3.3V
- **CLK**: 时钟信号，通过33Ω电阻连接
- **GND**: 接地
- **CMD**: 命令/响应信号，通过10KΩ上拉电阻连接3.3V
- **DAT0**: 数据位0，通过10KΩ上拉电阻连接3.3V
- **DAT1**: 数据位1，通过10KΩ上拉电阻连接3.3V
- **VDD**: 电源(3.3V)，连接2.2uF电容到地

### 5.3 SD总线模式协议

SD总线允许动态配置数据线的数量，从1到4条双向数据信号。开机后，SD卡默认仅使用DAT0。初始化后，主机可以更改总线宽度。

主机可以连接多个SD卡。在多个连接中，共享VDD(电源)、VSS(地)和CLK(时钟)信号连接。但是，从主机到每个设备，命令、响应和数据线(DAT0-DAT3)应分别分配。

此功能允许在硬件成本和系统性能之间进行轻松权衡。SD总线上的通信基于由起始位发起并由停止位终止的命令和数据位流。

**(1) 命令**
命令在CMD线上串行传输。主机给设备发的一个"开始行动"的命令。这个命令可以发给指定的某一张卡(这就是所谓的"寻址命令")，也可以发给所有已连接的卡(这就是"广播命令")。

**(2) 响应**
响应在CMD线上串行传输。响应就像是设备对之前收到的命令给出的一个"回音"或"答复"。这个答复可以是由指定的某一张卡发出的(也就是被寻址的那张卡)，也可以是由所有已连接的卡一起发出的。

**(3) 数据**
数据可以从卡传输到主机，反之亦然。数据通过数据线传输。

**SD NAND总线信号**:
- **CMD**: 双向命令/响应信号
- **DAT0-DAT3**: 4条双向数据信号
- **CLK**: 从主机到卡的时钟信号
- **VDD, VSS**: 电源和接地信号

SD NAND总线通过这些信号来实现主机与SD卡之间的通信和数据传输。CMD信号用于发送命令和接收响应，DAT0-DAT3信号用于数据的双向传输，CLK信号提供时钟同步，而VDD和VSS则分别提供电源和接地。

### 5.4 卡初始化

为了初始化SD NAND卡，以下是一个推荐的操作步骤示例：

1. **初始化供电电压**: 主机系统可以从初始化阶段开始为SD卡提供工作电压。向SD卡施加超过74个周期的虚拟时钟信号。

2. **选择操作模式(SD模式或SPI模式)**:
   - 如果采用SPI模式操作，主机应将SD卡接口的1个引脚(CD/DAT3)驱动到"低"电平。然后，发送CMD0命令。
   - 如果采用SD模式操作，主机应驱动或检测SD卡接口的1个引脚(该引脚的上拉寄存器通常被上拉到"高"电平)。
   - 除非重新发送CMD0命令或断电，否则卡将保持所选的操作模式。

**SD模式初始化步骤**:

3. 发送ACMD41命令，参数(Arg)设为0，并确定卡的工作电压范围。

4. 将指示的工作电压施加到卡上。重新发送带有应用电压存储的ACMD41命令，并重复发送ACMD41命令，直到忙碌位被清除(位31, Busy=1)。如果响应超时，主机可以识别出这不是SD卡。

5. 发送CMD2命令，并获取卡识别码(CID)。

6. 发送CMD3命令，并获取相对卡地址(RCA)。(RCA值在每次访问时随机变化，不为零)

7. 发送CMD7命令，并转移到传输状态。如果必要，主机可以发送ACMD42命令，并禁用用于卡检测的上拉电阻。

8. 发送ACMD13命令，并轮询SD存储卡的卡状态。检查SD_CARD_TYPE值。如果重要的8位都是"零"，则表示是SD卡。如果不是，则停止初始化。

9. 发送CMD7命令，并转移到待机状态。发送CMD9命令，并获取CSD(卡特定数据)。发送CMD10命令，并再次获取CID。

10. 使用CMD7命令返回到传输状态。

11. 发送ACMD6命令，并选择适当的总线宽度。然后，主机可以像访问存储设备一样访问SD卡中的数据。

### 6.2 启动

**启动示意图**

芯存SD NAND在上电并准备接受命令的过程中，电源供应需要满足的时间要求和信号状态。首先，电压需要在250毫秒内从0上升到VDD_min，然后主机需要开始发送至少74个SD时钟信号给SD卡，同时确保CMD线保持高电平状态，这样SD卡才能正确初始化并准备接受命令。

**启动时序要求**:
- 初始化延迟: 最大1毫秒，74个时钟周期和电源爬升时间
- 初始化过程超时值: 1秒
- 从第一次ACMD41到卡就绪的时间

### 6.3 上电复位电平

在上电爬升之前，主机需要保持电源线电平低于0.5V，并且持续时间超过1毫秒。

电源爬升时间是指从电压达到0.5V这个门槛开始，一直到电压稳定上升到设备正常工作的电源电压范围（这个范围在VDD的最小值和最大值之间），并且在这个时候，主机可以开始提供SD时钟信号（SDCLK）的时间段。

以下是关于电源爬升的推荐建议：
- 电源爬升过程中的电压应尽可能单调递增
- 电源爬升的最小时间应为0.1毫秒
- 对于2.7~3.6V的电源供应，电源爬升的最大时间应为35毫秒
- 主机在发出第一个命令前至少提供74个时钟周期

### 6.4 开机或电源循环

为了确保芯存SD卡能够可靠地进行硬重置，以下是开机和电源循环的要求：

1. 电压水平必须低于0.5V
2. 这个低电压状态需要保持至少1毫秒

当主机关闭电源时，VDD(设备供电电压)必须降低到0.5伏特以下，并且这个低电压状态至少要持续1毫秒。在关机过程中，主机应该将DAT(数据线)、CMD(命令线)和CLK(时钟线)断开，或者将它们驱动到逻辑0状态，以避免通过信号线产生工作电流。

如果主机需要改变工作电压，那么就需要进行电源循环。电源循环意味着先关闭电源，然后再重新供电。对于已经处于非活动状态的卡，也需要进行电源循环来访问它们。为了创建电源循环，主机应该先按照关机说明操作(即将VDD降低到0.5伏特以下，并保持至少1毫秒)，然后再给卡供电。
